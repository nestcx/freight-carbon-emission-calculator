Quick note on working with templates


All pages should extend the templates/base.html file. Sometimes though, you may need to insert a custom CSS or JS file specifically for the page you're working on. To do this, you can use one of the defined blocks


Referencing custom css files

To reference a CSS file that isn't the style.css file, use the code below in your html file:

{% raw %} {% block extra_css %}
code for inserting css file goes here
{% endblock %} {% endraw %}



Referencing custom JS files

To reference a custom JS file, add the code block below into your html file.
NOTE: Make sure that the inserted code block is just before the final {% raw %}{% endblock %}{% endraw %} that you find at the end of the html file, otherwise you will try to insert your script after the </html> block, which wouldn't work



{% raw %} {% block extra_js_scripts %}
Your JS script reference goes here
{% endblock %} {% endraw %}

Flashes

Flask allows you to flash a message via the backend to the frontend. This can be useful for things like providing feedback to the user if they coulnd't log in or enterred invalid input, etc. You can use Jinja to display these flash messages, and add class names so they're automatically styled based on the type of message


Feedback

Feedback will go in here

<div class="flash--feedback">
<p>Feedback</p>

Success

This will provided feedback indicating success

<div class="flash--success"> <p>Success</p>

Error

This will list errors

<div class="flash--error"> <p>Error</p>



Mixins

Mixins allow you to define styles that can easily be reused. We're not using this much for this project, but you should still have a look at the abstracts/mixins.scss page


To use a mixin, you type @include mixinname() { inside the current rule you're styling


Examples


@include border-shadow-soft()

@include border-shadow-large-example()


More importantly, we use mixins to easily write styles for larger sized screens. The default rules should be styled for mobile-sized viewports, while mixins should be used to write styles for larger screens. To do so, use the code:


@include breakpoint-screen-larger-than($breakpoint-tablet) {
styles for larger screens go here
}
